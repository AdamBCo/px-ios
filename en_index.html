1. Add our dependency to your project

If you use CocoaPods in your project, you can add the dependency in the Podfile.

If you don't use CocoaPods, download MercadoPago's SDK in this link and add this in your project.

                     
source 'https://github.com/CocoaPods/Specs.git'
# Se necesita este parámetro por ser una SDK en swift
use_frameworks!
platform :ios, '8.0'
pod 'MercadoPagoSDK', '2.2.1'

2. Set your public key

Set the public key corresponding to your MercadoPago account in the AppDelegate class. In this case we will use a test key. Later on we will see how you can get your own public key.

If your proyect is writtern in Objective-C you need to set in "Build Settings" the parameter "Embedded Content Contains Swift Code" in "Yes", because the SDK is written in Swift. 

               
                  class AppDelegate: UIResponder, UIApplicationDelegate {

                    var window: UIWindow?
                    // 1 - Publick Key
                    let publicKey = "TEST-ad365c37-8012-4014-84f5-6c895b3f8e0a"

                    func application(_ application: UIApplication,
                      didFinishLaunchingWithOptions launchOptions: [UIApplicationLaunchOptionsKey: Any]?) 
                        MercadoPagoContext.setPublicKey(publicKey)
                         // Set your country
                        MercadoPagoContext.setSite(MercadoPagoContext.Site.MLA)
                        return true
                    }
                  }

3. Add a payment button in your App

As an example, we are going to start the MercadoPago's flow from a button.

Create a ViewController (MainViewController, for example) and insert a button in the corresponding .xib. Also add a UILabel (we are going to call it paymentResult) to show the payment Results

Paste this code in MainViewController.swift.

In the next step, you will be working in the event associated to the button (startCheckout). 

               
                 import UIKit

                    class MainViewController: UIViewController {

                      @IBOutlet weak var payButton: UIButton!

                      @IBOutlet weak var paymentResult: UITextField!

                      override func viewDidLoad() {
                        super.viewDidLoad()
                        
                        self.payButton.addTarget(self, 
                          action: #selector(MainViewController.startCheckout), 
                          for: .touchUpInside)
                      }
                    }

4. Start the checkout from your app

Try it out with our info:
Use this code with our info to see a quick trial of the full experience.
When you want to go into production, you'll just have to use your info.

To start our checkout you just need:

  • Public Key: It's an identifier that groups your account, app and its configurations.
    • Create your credentials.
    • Configure your app.
  • Payment preference identifier: it's a representation of what you want to be paid for. It includes this attributes:
    • Info and amount.
    • Accepted payment methods.
    • Payer information (the email is mandatory for security reasons).
  • Optional: You can change the app color from here

               
                        // Preference Id
                        let prefId = "150216849-ceed1ee4-8ab9-4449-869f-f4a8565d386f" 

                        // Método ejecutado al hacer clic en el botón
                        func startCheckout(){
                          let checkout = MPFlowBuilder.startCheckoutViewController(prefId, 
                          callback: { (payment: Payment) in
                              // Done! The payment was processed by MercadoPago
                              self.displayPaymentInfo(payment)

                          }, callbackCancel: {
                              // User didn't finish the payment process
                          })

                          self.presentViewController(checkout, animated: true, completion: {})
                        }

5. Wait for the payment result

The SDK will always return a payment result

If an insurmountable error occurs or the user cancels the operation, it will return an exception so you know what happened.

These are the most important attributes of the payment object:

  • id: payment identifier.
  • status: payment status.
  • payment_method_id: an identifier of the payment method that your user selected.
  • payment_type_id: selected payment method type.
  • card: object that identifies your user's card.
  • issuer_id: an identifier of your user's selected card issuer.
  • installments: amount of installments selected.

Know more about the Payment objecthere. 

               
                  func displayPaymentInfo(payment : Payment){
                      let paymentInfo = "id : " + String(payment._id) + ", status : " + payment.status + ", status detail : " + payment.statusDetail
                      self.paymentResult.text = paymentInfo
                  }

7. Go into production

First:

  • Register a MercadoPago account to receive money.
  • Create an application.
  • Configure your credentials.

From your server:
Just create a payment preference and return our services response. Do it from your server, because you will have to sign it with your private key. By doing this we protect the payer and also your own seller user. How to create a preference in your server.

Complete preferences perform much better.
Send us all the information you can, and you will offer such an amazing experience for your buyers that your accredited payments will raise!

From your app:
We offer you a class called MerchantServer that connects with your server. The createPreference method makes a POST and sends as the message's body the map you've defined (preferenceMap). Set your base URL (https://api.tunombre.com) and URI (/create_preference) where you expect to receive the information in order to create a preference.
MerchantServer will turn your service's response (the same from MercadoPago services) into a CheckoutPreference object, which ID is the entry point to our checkout.
